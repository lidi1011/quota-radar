# Phase 1 Plan: Initial Release

This is a retrospective plan created after implementation so that v1.0.0 can be reviewed and closed through GSD.

## Goal

Ship Quota Radar v1.0.0: a normal macOS SwiftUI app that displays Codex and GLM / ZAI coding-plan quota status from local and provider-owned data sources.

## Scope

- Create SwiftPM macOS app scaffold and normal Dock-app window behavior.
- Build provider abstraction and first providers for Codex and GLM.
- Implement Codex local data collection from app-server rate limits, SQLite rollout paths, and session JSONL token events.
- Implement GLM quota API collection with settings/env credential resolution.
- Build responsive provider panels, quota rings, usage cards, wool progress card, settings tabs, and app icon.
- Add verification entry points: `swift test`, `swift build`, `./script/build_and_run.sh --verify`.
- Optimize Codex refresh performance by avoiding repeated full JSONL reads.

## Acceptance Criteria

- [x] Quota Radar launches as a normal macOS app with Dock icon and resizable standard window.
- [x] Codex and GLM provider panels render with rings, cards, refresh controls, and error states.
- [x] Settings support refresh interval, provider colors, card visibility, and GLM credentials.
- [x] Codex quota and token/cost calculations follow codexU/CodexBar-informed data sources and scope.
- [x] GLM quota API path follows glm-plan-usage-informed quota API behavior without requiring local dependency installation.
- [x] Tests, build, and App launch verification pass.

## Verification Commands

- `swift test`
- `swift build`
- `./script/build_and_run.sh --verify`

## Notes

- GLM real-account App validation remains dependent on user-provided GLM / ZAI credentials.
- This plan records completed work; future work should use normal discuss/plan/execute flow before implementation.
