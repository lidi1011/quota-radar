# .planning/matt

Matt Pocock skills 适配目录。

## 应该放什么

- `/grill-with-docs` 的追问结论、开放问题和领域建模记录。
- `domain-modeling` 产生或更新的术语、边界、命名约定和上下文摘要。
- `codebase-design` 与 `/improve-codebase-architecture` 产生的架构加深候选、选择理由和后续动作。
- 临时探索和候选可以留在这里；长期有效的关键决策应同步到 `docs/decisions/`。

## 给人类的阅读建议

这里记录的是“为什么这样理解问题”和“为什么这样设计边界”。如果要快速接手，先看 `.planning/HANDOFF.md`，再看本目录里最近的追问或架构记录。

## 给 agent 的执行建议

使用 Matt skills 时，不要让它替代 GSD 主流程。把可执行结论同步回 GSD 的 SPEC、CONTEXT、PLAN 或 `.planning/HANDOFF.md`，把长期决策写入 `docs/decisions/`。
