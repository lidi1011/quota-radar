# .planning/product-design

Product Design 工作流适配目录。

## 应该放什么

- `get-context` 确认后的产品、视觉和交互 brief。
- `research` 的用户痛点、UX 摩擦、竞品或来源证据摘要。
- `audit` 产生的 UX、设计和可访问性问题清单。
- `ideate` 的视觉方向选择记录。
- `prototype`、`image-to-code`、`url-to-code`、`design-qa` 的交付前检查和结论。

## 硬规则

没有 URL、截图、Figma、mockup、参考图或现有代码目标时，不直接开工做 UI 或原型。先走 Product Design 的 `get-context`，再用 `ideate` 生成三个视觉方向，等人类选定后再进入 prototype 或实现。

## 给 agent 的执行建议

Product Design 负责体验层，不替代 GSD 的项目生命周期。设计研究、审计和原型结论应回流到 GSD 的 SPEC、CONTEXT、PLAN、UI-SPEC 或 `.planning/HANDOFF.md`。截图、原型报告和可视化产物放到 `artifacts/product-design/`。
