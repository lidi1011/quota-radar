# scripts

可重复执行脚本目录。

## 应该放什么

- 放启动、测试、构建、迁移、导入导出、部署前检查等脚本。
- 脚本名称应清楚，例如 dev.sh、test.sh、seed-db.sh、export-report.sh。
- 人类应该能通过脚本减少手动步骤。
- agent 应优先调用已有脚本，而不是临时拼接复杂命令。

## 给人类的阅读建议

先看本文件了解边界，再进入具体文件。目录内如果出现难以归类的内容，应优先移动到更合适的位置，而不是继续堆积。

## 给 agent 的执行建议

修改本目录前先确认上游入口、下游调用和验证命令。完成后把重要结论写回 `.planning/HANDOFF.md` 或相关文档。
